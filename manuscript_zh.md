# 组合式记忆系统：认知访问的算子分解

> 日期：2026-05-24

###### 摘要

记忆增强型大语言模型（LLM）智能体传统上依赖检索——基于查询条件在记忆库中进行选择——作为主要的访问机制。本文认为，检索并非唯一根本机制，而是至少三种结构性不同的认知算子之一：检索（无序记忆库上的Top-K选择）、状态查寻（对压缩信念状态进行查询无关的读出），以及轨迹遍历（对时间有序转移图进行时序游走）。每种算子保留不同的认知维度并舍弃其余，使得结构上不同类型的记忆失效不可避免；没有单一算子是完备的。我们提出一种基于算子组合的记忆系统设计方法论：将任务空间分解为认知轴，通过经验测量的相图选择各轴的最优算子，针对目标基座模型进行校准，并通过类型安全的路由进行算子组合。四个LLM上的跨模型验证表明，算子互补性是模型无关的——状态查寻在全部四个模型的112个案例中均有相同的29个案例失败，且输出几乎逐字相同的错误答案——而最优路由则是模型条件化的。遵循模型校准路由的组合系统在三个基准测试（MSFT、MSFT-NL、STALE）上相比最佳单算子实现了+0.038至+0.138的增益。关键的是，经验结果表明，校准后的组合在所有测试设置下均表现出非劣性。核心启示是：记忆系统设计应被重新定义为含模型专属校准的算子组合，而非单一访问范式下的表示优化。

## 1 引言

长时程运行的大语言模型（LLM）智能体依赖有效记忆——即访问、推理并维护过去信息的能力。主流方法是检索增强型记忆：过往交互被存储为外部记忆片段，在查询时选择性地检索。近期研究推动了记忆表示（APEX-MEM的时间属性图和过程性学习、GSW的生成式语义工作空间、GAM的层级式图整合）、自适应检索（UMA基于强化学习的CRUD记忆管理），以及记忆评估（AMA-Bench的长周期评估、ReMemR1的检索循环式记忆更新）的发展。然而，这些工作共享一个共同假设：记忆是记忆片段的集合，智能主要体现在于检索时的选择和下游推理。

近期的证据开始挑战这一基础。在多个长期记忆基准测试中，最先进的记忆智能体相较于基础检索仅带来边际改进，且频繁复用无效或过时的记忆（如 Memora（Uddin 等人，2026）中报告的 LoCoMo/LongMemEval 基准测试结果）；STALE（Chao 等人，2026）进一步证明，即便更新后的证据被显式检索到，系统也常常未能将其视为权威——尽管更新后的证据出现在排名靠前的检索结果中，仍有56.1%的案例表现出残余行为失效。这些发现汇聚为一个结构性诊断：记忆失效主要不是检索问题，而是未能随时间推移维持一个连贯且权威的状态。更深层的问题——什么样的访问机制最能服务于智能体的认知需求——仍悬而未决。

本文认为，检索与状态之间看似二元的选择是一个错误二分法。至少存在三种结构性不同的认知算子：检索（基于查询条件在无序记忆库上进行选择）、状态查寻（查询无关地读出压缩信念状态），以及轨迹遍历（对时间有序转移图进行时序游走）。每种算子保留一组不同的认知维度并舍弃其余。因此，没有单一算子是完备的——每种算子都使特定类型的记忆失效在结构上不可避免。设计问题更应被理解为：由任务认知空间的结构所引导的访问算子组合，而算子的有效性则进一步受基座模型选择的制约。

**贡献一：为何单一算子不够**
我们定义了三种认知算子，并通过受控基准证明，每种算子在共享的记忆轨迹上均会引发一种独特且不可恢复的投影失效。检索使矛盾在结构上变得不可观测。状态查寻通过将记忆坍缩为单一值表示来消除这种不稳定性，却使历史查询和关系查询在结构上变得不可提问。轨迹遍历保留了完整的时间记录，避免了上述两种失效，但对于决策时推理而言是不可压缩的。没有任何单一投影能同时保留共现性、时间访问和可压缩性。在四个基座大语言模型上，状态查寻表现出近乎一致的失效模式，而检索与轨迹的偏好在不同模型间存在显著偏移——算子互补性是模型无关的；最优路由则是模型条件化的。

**贡献二：一种组合方法论**
我们提出一个四步方法论：将任务空间分解为认知轴，查阅算子相图为各轴选择初始算子，针对目标基座模型校准选择，并通过类型安全的路由进行算子组合。相图提供初始假设；模型专属校准确认或修正之——在任何测试设置下，校准后的组合均不劣于最佳单算子。

**贡献三：多尺度的经验验证** 遵循模型校准路由的组合系统在结构化基准（MSFT）、自然语言基准（MSFT-NL）和外部冲突基准（STALE）上，均取得了相对于最佳单算子的一致的路由增益。关键的是，经验结果表明，校准后的组合在所有经测试的模型-基准组合上均保持非劣性。这些增益，以及非劣性这一性质，在四个基座大语言模型上持续成立，确认了该方法论跨模型泛化的能力。在大规模真实世界基准 ATM-Bench 上，算子层面的分析揭示了不同的认知覆盖区域，其中有相当比例的查询需要超出本文所研究的三种算子之外的算子类别——组合框架的设计正是为了容纳这一方向。

## 2 三种认知算子

三种访问机制的数学结构各异。检索是连续空间上的相似度匹配；状态查寻是键值空间上的离散分类；轨迹遍历是时间有序图上的确定性游走。每种机制引入一个独特的认知算子，具有特定的结构属性。
### 2.1 形式定义

我们将智能体记忆建模为具有时间邻接关系的事件序列：

$$T = \{e_1, e_2, \dots, e_n\}, \quad e_{i} \prec e_{i+1}$$

其中$\prec$表示时间先后。从这一原始对象，我们推导出两种降维表示：

状态投影： $\mathrm{proj}_s(T) = s_n$——当前信念状态，不保留达成信念的过程信息。
记忆库： $M_T = \{e_1, \dots, e_n\}$——无序证据片段集合，剥离了时间顺序和因果邻接关系。
转移图： $G_T = (V, E)$，其中 $V = \{e_1, \dots, e_n\}$，$E = \{(e_i, e_{i+1}) \mid 1 \leq i < n\}$——编码连续事件之间时间邻接关系的有向图。
每种表示引入一种认知算子：

| 算子 | 表示 | 上下文 | 选择机制 |
|---|---|---|---|
| 检索（$\mathcal{R}_T$） | 记忆库 $M_T$ | 按相似度排序的子集 | 查询时（$\text{top-}K_\theta(M_T, q)$） |
| 状态（$\mathcal{S}_T$） | 状态投影 $\mathrm{proj}_s(T)$ | 所有活跃键值对 | 写入时（经修订 $U$） |
| 轨迹（$\mathcal{T}_T$） | 转移图 $G_T$ | 有序子图 | 查询时（确定性游走） |

### 2.2 每种算子保留与舍弃的内容

| 算子 | 保留 | 舍弃 |
|---|---|---|
| $\mathcal{R}_T$ | 原始内容（词汇/语义重叠） | 状态语义，时间顺序 |
| $\mathcal{S}_T$ | 当前信念（事实一致性） | 时间/因果结构，替代选项 |
| $\mathcal{T}_T$ | 时间顺序，状态演化 | 紧凑性，查询独立性 |

这些保留与舍弃的模式决定每种算子的结构性失效区域。$\mathcal{R}_T$无法解决写入时冲突，因为它从不将冲突证据聚合为单一状态。$\mathcal{S}_T$无法回答历史查询（"最初的预算是多少？"），因为它在构造时即丢弃了轨迹，每个键仅保留最新值。$\mathcal{T}_T$无法高效提供简洁的状态摘要，因为它保留了完整的时间序列。没有算子能保留所有认知维度；因此没有单算子是完备的。

算子与存储表示之间的区分至关重要。本文所定义的算子刻画的是记忆如何被访问，而非如何被存储。向量数据库、文件系统层级、SQL表、知识图谱以及语义工作空间属于表示层的选择：它们决定信息如何被编码、组织、索引或持久化。而本文研究的认知算子刻画的是施加于这些表示之上的访问机制——查询条件化选择（$\mathcal{R}_T$）、查询无关的状态读出（$\mathcal{S}_T$），或者按时间顺序的遍历（$\mathcal{T}_T$）。表示决定存储拓扑；算子决定查询时的认知可及性。因此，更丰富的表示并不会消除算子之间的区分：同一个基于图的记忆系统，取决于访问方式的不同，仍可能以检索、状态查寻或轨迹遍历的方式运作。因此，我们的主张关乎访问原语，而非存储格式。

我们不声称这三种算子构成记忆访问机制的完备本体。其他超出本文研究范围的访问机制可能实例化为不同的认知算子，特别是那些引入可学习的递归、结构化的潜在状态演化或程序化记忆控制的机制。我们的主张更为限定：这三种算子代表了在当前记忆增强型LLM系统中经验上占主导地位的不可约的访问原语，且它们的互补性已使算子组合成为必要的设计考量。

核心启示并非某一算子更优——最优算子取决于查询的结构——而是算子选择是记忆系统架构中一个不可约的设计决策。本文后续部分形式化每种算子在何时合适，以及如何组合它们。

## 3 相关工作

**当前的记忆系统是单算子实现。** 绝大多数记忆增强型LLM系统实现了$\mathcal{R}_T$：带查询条件化检索的记忆片段存储，仅在存储结构和检索启发式上有所不同。LightMem（Fang等人，2026，ICLR 2026）提出含离线整合的三阶段记忆，实现了显著的效率提升，但其访问机制仍是查询条件化检索。基于图的系统，如APEX-MEM（含多工具检索的时间属性图，ACL 2026）和GSW（生成式语义工作空间，AAAI 2026口头报告），将记忆片段组织为结构化图，但仍通过查询条件化选择进行操作。自适应检索系统——UMA（记忆CRUD与QA统一的端到端RL），ReMemR1（检索循环式记忆更新）——改进了选择机制，但仍在检索范式内。GAM（层级式图整合）提出结构化整合而非检索优化。另一条产业系统并行路线——OpenViking（字节跳动，2026）采用分级目录，Anthropic CMA（2026）采用可挂载记忆存储，Google Memory Bank（Vertex AI，2026）作为托管向量服务——正向结构化、类文件系统存储汇聚，代表了向$\mathcal{S}_T$式访问的转向。我们使用OpenViking对齐实现的受控实验证实，结构化检索优于扁平检索（废弃率0.24 vs 0.57），但撞上结构性天花板：任何检索变体都无法消除废弃值的使用，因为检索作为一个算子类别无法解决写入时冲突。

**近期研究挑战检索充分性。** STALE（Chao等人，2026）提供了最清晰的外部证据：即便更新后的证据在检索结果中明显可见，其仅被5.2%的情况视为权威，造成持久的行为失效。他们的CUPMem原型证实，写入时状态裁决可大幅提升性能（相同基座下从8.7%提升至68.0%）。STALE识别了记忆失效的经验症状；我们的算子框架提供了结构性解释——为何检索失效，状态何时足够，以及为何组合是必要的。DeMem（Zou等人，2026）提出决策中心遗忘标准。因果记忆干预（Srivastava，2026）将记忆选择视为因果推断问题，选择因果上有用而非仅主题相关的记忆——这是一种与我们写入时修订算子互补的查询时方法。

**本文定位。** 先前关于记忆增强型LLM系统的工作大多在单一算子类别内优化——改进检索排序、丰富状态表示或精炼轨迹编码。本文研究算子选择与组合：给定一组根本不同的访问机制，系统设计者应如何在其中选择，又如何将它们组合？算子组合框架运作于访问层——如何检索、投影或遍历已存储的表示。表示层上的互补工作，以STALE的潜在状态追踪为代表，处理的是应存储什么。两者独立：更丰富的表示缩小个体算子的失效区域，但不消除算子组合的结构必要性，因为没有单一访问机制能同时服务于检索广度、状态一致性和时间保真度。算子组合是与表示工程处于不同层面的设计决策。

# 4 为何单一算子不够

### 4.1 记忆访问作为认知投影

我们将智能体记忆建模为一条轨迹——一个按时间排序的事件序列（第2.1节）。每种记忆访问算子对应于该轨迹向低维子空间的一次投影，保留某些认知维度而舍弃其余。$\mathcal{R}_T$投影到相似度几何上；$\mathcal{S}_T$投影到最新值压缩上；$\mathcal{T}_T$投影到时间排序上。没有单一投影能保留所有维度。每种算子定义了一个独特的信息保留约束系统，而非一种检索策略。问题在于所导致的信息损失能否通过下游推理来恢复——我们的结果表明不能。

### 4.2 检索：通过结构性选择失效导致的认知不可见性

$\mathcal{R}_T$引入了一种表示体制，在此体制中，矛盾在任何上下文规模下都系统性地不可观测。这不是关于性能的声明——而是关于该算子使什么东西在结构上不可能被看到。一个受控证据累积实验（9步轨迹：3步一致，4步冲突，2步澄清性事实更新）揭示了这一机制。

**共现性坍缩 → 认知湮灭。** 在累积协议的所有步骤中，正确证据与过时证据从未同时出现在top-$k$检索输出中（共现性 = 0.00）。检索运作于一种矛盾证据在结构上无法共现的体制中，使得矛盾变得不可表示，而非未被解决。LLM看到的是一个内部一致但在时间上过时的记忆视图，且无法感知该矛盾。矛盾感知在所有步骤中始终为零。

**排序反转 → 度量诱导的认知扭曲。** 第9步的单轨迹追踪揭示了这一机制。对于查询"Alice现在住在哪里？"，检索排序如下：

| 排名 | 证据片段 | 相似度 |
|:----:|--------|:------:|
| #1 | Alice住在西雅图。 | 0.125 |
| #2 | Alice是一名软件工程师。 | 0.100 |
| #3 | Alice每天骑自行车通勤。 | 0.100 |
| #4 | Alice辞去了工程师工作，成为了一名设计师。 | 0.083 |
| #5 | Alice搬到了芝加哥。← *正确* | 0.077 |
| #6–#7 | （关于其他实体的证据：Bob、Charlie；为清晰起见省略） | 0.045–0.056 |
| **#8** | **Alice不再住在西雅图。她已于2025年永久迁至芝加哥。← 显式更正** | **0.038** |

轨迹中最具权威性的证据——一条显式更正——排名第八，位于四条过时片段之后。该更正与查询共享相同的词汇重叠（{alice}），但包含18个额外词汇，这些词汇使Jaccard分母扩大约三倍，却未贡献任何额外的重叠，致使其得分低于更短、信息稀疏的过时片段。这种排序惩罚是结构性的：相似度度量以一种系统性压制更正性证据的方式重新定义了认知显著性。系统偏向的不是错误答案，而是认知上不完整的证据。

**单调退化 → 渐进性认知漂移。** 在$\mathcal{R}_T$下，过时答案率从0.00单调上升至第9步的0.57。仅追加存储与相似度排序相结合，引发了一条朝过时不动点渐进认知漂移的轨迹。

**系统性不稳定。** 在四条独立构造且证据结构相同的轨迹上，$\mathcal{R}_T$在第9步的过时率范围为0.286至0.571（0.39±0.14），而$\mathcal{S}_T$收敛至0.00±0.00（附录H.1）。2倍差异并非噪声，而是结构化的变异：检索排序取决于查询-证据的词汇对齐，而词汇对齐因领域而异，且不在系统设计者的控制范围内。$\mathcal{R}_T$表现为一个没有固定认知点的随机采样器。

**架构天花板 → 在检索设计类下不变。** 这一天花板在不同检索架构下持续存在。一个文件系统范式的检索器（OpenViking对齐，含L0/L2检索的分层目录）将过时率从0.57降至0.24；增加LLM增强的L0生成和查询改写进一步将其降至0.15——但这一失效模式在检索设计类下是不变的。没有任何检索变体能消除过时值的使用。$\mathcal{S}_T$配合写入时修订始终保持0.00。

**因果归因。** 反事实设计——相同的轨迹、相同的证据、相同的查询、相同的模型，仅区别在于是否应用修订算子$U$——确立了该差距（0.57 vs 0.00）在因果上可归因于算子选择。$\mathcal{R}_T$无法维持一致的状态，因为它根本没有状态可维持；$\mathcal{S}_T$可以，因为它将写入时冲突解决与读取时访问相分离。这不是实现上的差异——而是算子定义的结构性后果。

$\mathcal{R}_T$并非未能表示冲突；它定义了一种注意力几何，在此几何中，冲突证据无法进入共享的表示子空间。矛盾并未丢失；它从未被共同实例化。

### 4.3 状态查寻：通过表示坍缩实现认知确定性

$\mathcal{S}_T$通过将记忆坍缩为单值认知区域来消除检索引发的冲突。这不是检索的改进——而是一种不同的认知体制。$\mathcal{S}_T$并不返回关于可能状态的不确定分布；它返回单一承诺值。它以表示的确定性取代了认知不确定性。这引发了一种认知不对称性：被表示的内容可以确定地知晓，而被丢弃的内容即使在完美的下游推理下也变得不可恢复。这种损失在任何下游推理下都是不可逆的，因为缺失的表示维度永远不会在输入空间中被实例化。

**状态查寻解决了什么。** 在相同的9步冲突累积协议下，$\mathcal{S}_T$在所有四条轨迹上收敛至0.00过时率（第4.2节）。其优势并不依赖于人工标注的输入：当状态提取完全由LLM从非结构化文本中执行时，过时率为3.85%，而扁平检索为19.23%——提升5倍。提取误差有界（约0.04），不随记忆体量增大而复合；检索误差无界，因为排序竞争随记忆库增长而加剧。这种稳定性是通过在构造时消除表示多重性来实现的，使不确定性变得不可表示，而非未被解决。其结果是，任何时间或关系查询都在结构上被投影到了表示空间之外。

**状态查寻使什么变得不可提问。** MSFT逐模式细分揭示了$\mathcal{S}_T$在何处结构性失效（完整表格见附录B.1）：

| 模式 | n | $\mathcal{R}_T$ | $\mathcal{S}_T$ | $\mathcal{T}_T$ | 结构根源 |
|---|---|---|---|---|---|
| 历史状态 | 12 | **1.000** | 0.167 | 0.750 | $\mathcal{S}_T$丢弃历史 |
| 跨实体 | 16 | 0.500 | **0.125** | 0.625 | $\mathcal{S}_T$无法对齐时间轴 |
| 时间关系 | 12 | 0.667 | **0.000** | 0.750 | $\mathcal{S}_T$无时间表示 |
| 更正链 | 12 | 0.750 | **0.917** | 0.833 | $\mathcal{S}_T$坍缩为终值 |
| 多路径 | 12 | **0.917** | 0.333 | 0.750 | $\mathcal{S}_T$覆写分支值 |
| 模糊 | 24 | 0.500 | 0.500 | 0.500 | 所有算子均缺乏不确定性机制 |
| 负面证据 | 12 | 0.667 | 0.500 | 0.583 | — |
| 隐式冲突 | 12 | 0.667 | 0.500 | 0.667 | — |
| **总计** | **112** | **0.679** | **0.384** | **0.661** | 最佳单算子$\mathcal{R}_T$为0.679 |

$\mathcal{S}_T$不仅在历史查询（历史状态：0.167）、跨实体时间对齐（跨实体：0.125）和时间推理（时间关系：0.000）上表现不佳——它使这些查询类别在其表示空间内变得在结构上不可提问。它在压缩有益的地方取得成功：更正链（0.917），此处坍缩到最新值正是正确的操作。

**跨模型不变性。** 为区分算子固有失效与模型特异的弱点，我们在四个LLM上使用相同协议评估所有三种算子。$\mathcal{S}_T$在112个案例中有29个在所有四个模型上全部失效——25.9%的案例在结构上对状态查寻不可访问，无论模型家族或能力如何（附录E.7）。失效集中在历史状态（9/12，75%）和跨实体（7/16，43.8%）。关键的是，错误答案在不同模型间几乎逐字相同：在历史状态查询上，四个模型均回答当前值而非历史值。这些失效是模型无关的，因为信息损失是在表示构造时强加的，而非在推理时。相比之下，$\mathcal{R}_T$和$\mathcal{T}_T$的失效在很大程度上是模型依赖的：每种算子仅112个案例中的4个在所有模型上失败。$\mathcal{S}_T$是**表示受限**的；$\mathcal{R}_T$是**几何受限**的。

$\mathcal{S}_T$通过抹除替代选项来提供确定性。其优势即其弱点：消除冲突的同一投影，也消除了提出某些问题的能力。

### 4.4 轨迹遍历：无压缩性的认知完备性

$\mathcal{T}_T$保留了完整的认知痕迹，但违背了决策时推理所需的表示可压缩性。它是唯一在没有投影引发损失的情况下保留认知历史的算子，但代价是对下游决策而言不可压缩。在路径依赖的模式上，它在两个替代方案均挣扎的地方表现出色：时间关系（0.750）、多路径（0.750）、跨实体（0.625）。这些查询需要追踪一系列状态变化——这是一种基于独立片段的检索和基于坍缩值的状态查寻所无法执行的操作。然而，在更正链（0.833）上，$\mathcal{T}_T$提供了完整的事件流，而单一键值读出（$\mathcal{S}_T$，0.917）即已足够。轨迹算子是最完整的时间见证者，但最不可压缩为决策就绪的形式。$\mathcal{T}_T$是唯一在原则上保留认知完备性的算子，但无法将其归约为有界的决策表示。

### 4.5 从三难困境到结构不相容性

三种算子定义了对共享认知空间而言彼此互斥的约束。没有任何投影能同时保留冲突证据的共现性、时间结构和决策时可压缩性。

$\mathcal{R}_T$引发**可见性坍缩**：基于相似度的选择在结构上压制更长、更具信息量的更正性证据，使矛盾在任何上下文规模下均不可观测。冲突未被解决——它被排除在表示空间之外。

$\mathcal{S}_T$引发**时间坍缩**：通过将完整轨迹投影到最新值压缩上，它消除了表示多重性，随之也消除了回答历史或跨实体查询的任何能力。一致性通过抹除而非解决来实现。

$\mathcal{T}_T$引发**压缩坍缩**：通过保留完整的时间序列，它维持了对所有证据的访问，但牺牲了紧凑性和查询独立性，使其成为最完备但最低效的算子。

这些坍缩模式是不可约的：$\mathcal{R}_T$在历史状态上成功（1.000），而$\mathcal{S}_T$失败（0.167）；$\mathcal{S}_T$在更正链上成功（0.917），而$\mathcal{R}_T$较弱（0.750）；$\mathcal{T}_T$在时间访问上占主导地位，而两个替代方案在结构上均无法胜任。每种模式的最优算子在八种模式中的七种上因模型而异（附录E.7）。

任何单一算子都会引发一个结构上不可避免的认知盲区，该盲区无法通过在推理时进行组合来恢复。记忆推理不能被归约为任何单一算子而不产生认知维度的结构性损失。这不是工程的失败，而是**单一投影记忆访问下的表示不可能性**。单一算子的记忆访问并非一个可通过渐进式改进来优化的次优选择——每种投影都会永久性地禁用至少一个认知维度。因此，记忆推理不是一个算子选择问题，而是一个从互不相容的投影中重建完整认知状态的问题。这必然要求动态组合：一个系统必须选择、组合并校准互补算子，以覆盖不可约的认知空间。下一节将发展操作化这一原则的组合方法论。

## 5 基于不相容投影的组合

### 5.1 从结构不相容性到组合必要性

第4节确立了三种算子各自引发一个结构上不可避免的认知盲区，且这些盲区是互补的而非重叠的。剩下的问题不是选择哪个算子，而是如何从共享记忆轨迹上的互补投影中恢复可用的认知覆盖。组合之所以必要，是因为在一个投影下缺失的信息在该投影的表示空间内不可恢复。覆盖只能通过组合那些保留互补认知维度的算子来重建。三种算子与五个认知轴（时间访问、冲突解决、实体组合、认知不确定性、反事实推理）相互作用，形成一个3×5的投影兼容性交互空间。我们对该空间进行经验刻画，以识别每种算子为各认知轴保留了所需的认知结构。

### 5.2 相图作为经验刻画

相图是一张经验性的图谱，描绘了在每个投影算子下哪些认知体制仍可表示。它不是算子性能的排行榜，而是跨认知体制的投影兼容性刻画。

| 认知轴 | $\mathcal{R}_T$ | $\mathcal{S}_T$ | $\mathcal{T}_T$ | 兼容投影 |
|---|---|---|---|---|
| 时间访问 | 0.617 | 0.433 | **0.700** | $\mathcal{T}_T$（有序遍历） |
| 冲突解决 | 0.817 | **0.950** | **0.967** | $\mathcal{S}_T \approx \mathcal{T}_T$（压缩或时间支持） |
| 实体组合 | **0.533** | 0.700 | 0.650 | $\mathcal{S}_T > \mathcal{T}_T$（关系绑定偏好压缩） |
| 认知不确定性 | 0.550 | **1.000** | 0.550 | $\mathcal{S}_T$（压缩消除歧义） |
| 反事实推理 | 0.900 | 0.817 | **1.000** | $\mathcal{T}_T$（时间记录保留条件性） |

每个认知轴偏好不同的算子，因为每种算子保留不同的认知维度（第2.2节）：$\mathcal{T}_T$保留时间顺序，$\mathcal{S}_T$通过压缩保留事实一致性，$\mathcal{R}_T$保留原始内容但舍弃结构。在冲突解决上，$\mathcal{S}_T$和$\mathcal{T}_T$均成功，因为它们都能化解矛盾——$\mathcal{S}_T$通过写入时压缩，坍缩为最新值；$\mathcal{T}_T$通过时间支撑，呈现出完整的解决链条。这些主导模式共同构成一张兼容性图谱：哪个投影与哪种认知需求在结构上对齐。

跨域对比确认了这一兼容性结构在结构化和自然语言输入下的稳定性：

| 算子 | 结构化 | 自然语言 | 增量 | 属性 |
|---|---|---|---|---|
| $\mathcal{R}_T$ | 0.680 | 0.683 | +0.003 | 域不变 |
| $\mathcal{S}_T$ | 0.500 | 0.780 | +0.280 | 压缩 |
| $\mathcal{T}_T$ | 0.880 | 0.773 | −0.107 | 有序访问 |

$\mathcal{R}_T$是域不变的：其性能取决于查询与记忆表层形式之间的相似度几何，与形式的组织方式无关。$\mathcal{S}_T$表现出压缩行为：自然语言提供了隐式规范化，改善了信息提取。$\mathcal{T}_T$是有序访问的：其有效性依赖显式时间顺序，当需要从自然语言推断顺序时性能下降。跨模型验证（GPT-4o-mini）表明，仅压缩属性是模型无关的；域不变和有序访问属性随模型能力变化，这强化了模型校准的必要性（第5.3节，第2a步）。

### 5.3 组合方法论

相图（第5.2节）为一种组合方法论提供了经验基础，该方法论从结构上不完备的算子中恢复认知覆盖。该方法论并不优化任何单一算子；它通过将每个查询派发到其投影与该查询的认知需求在结构上兼容的算子，来防止认知上不兼容的算子-查询配对。

**第一步：分解任务空间。** 识别目标任务中存在的认知轴——时间访问、冲突解决、实体组合、认知不确定性、反事实推理，或特定于领域的其他轴。每个轴对应一个独特的认知操作，可能需要不同的访问机制。在本文中，任务分解由作者根据各基准的结构手动完成；自动化或标准化此步骤是第8节讨论的一个开放问题。

**第二步：将每个轴映射到兼容的投影，并针对目标模型进行校准。** 相图（第5.2节）将每个认知轴分配给保留其所需结构的投影：时间访问 → $\mathcal{T}_T$，认知不确定性 → $\mathcal{S}_T$，实体组合 → $\mathcal{S}_T$（避免$\mathcal{R}_T$），冲突解决 → $\mathcal{S}_T$或$\mathcal{T}_T$，反事实推理 → $\mathcal{T}_T$。这一映射识别出结构上兼容的算子-查询配对。相图是在单一基座模型上测得的；算子之间的确切主导边界可能随模型变化而发生偏移——在DeepSeek Flash上，算子表现均衡，路由增益较大；在GPT-4o-mini上，$\mathcal{T}_T$占主导地位，增益较小。为应对这一点，在目标模型下通过小型诊断样本（第4节）测量算子准确率，并据此调整映射。相图提供结构先验；校准则修正基座依赖的偏移。

**第三步：通过类型安全的路由进行组合。** 实现一个路由器，将每个查询分类到其认知轴，并派发到其投影与该认知轴兼容的算子。路由架构是类型安全的（一个查询，一个算子）以避免融合歧义。组合提升性能并非通过增强任何单一算子，而是通过防止结构上不匹配的算子-查询配对。前端分类器是可替换组件；算子层是稳定的（第4节）。

**第四步：对照单算子基线验证。** 在目标基准上对比组合系统与每个单算子。若组合未能提升对最佳单算子的覆盖，则任务空间可能过于同质而无需组合，或者相态校准需要调整。在所有经测试的基准和基座模型上，校准后的组合一致地恢复了任何单一算子无法访问的任务区域（第6节）。

下一节在三个基准和四个基座模型上验证该方法论。

## 6 验证基于不相容投影的组合

第4节确立了单一算子会引发不可约的盲区。第5节提出了基于不相容投影的组合作为结构性回应。本节从四个问题出发验证该回应：组合是否能恢复覆盖（第6.1节），方法论是否能跨基准迁移（第6.2节），当前算子基在何处触及极限（第6.3节），以及校准是必要的而非可选的（第6.4节）。

### 6.1 组合下的覆盖恢复

**组合能否恢复任何单一算子无法访问的任务区域？** 我们将第5.3节的方法论应用于112案例的MSFT基准。三种算子及组合系统使用相同的基座LLM（DeepSeek Flash，温度0.0）、相同的证据和相同的二元裁判。

在112案例的MSFT基准上，组合系统达到0.817，而最佳单算子（$\mathcal{R}_T$）为0.679——路由增益为+0.138。算子层未产生任何新的失效：全部19次残余失效均源自意图检测器前端（10次召回错误）或已知的算子边界（6个跨实体案例）。完美路由的理论上限为0.913，确认路由架构并非瓶颈——前端分类器是可替换组件。组合提升性能并非通过增强任何单一算子，而是通过防止结构上不匹配的算子-查询配对。

路由增益并非集成效应：每个算子独立处理其分配的查询类型，没有查询被多于一个算子处理。增益源于将每个查询匹配到其投影保留了该查询所需认知维度的算子——而非源于对多个输出的统计聚合。

**跨模型非劣性。** 在GPT-4o-mini上，针对其逐模式算子偏好校准的组合系统达到0.830，而最佳单算子为0.812（+0.018）。尽管增益小于DeepSeek Flash上的（+0.138）——归因于GPT-4o-mini的$\mathcal{T}_T$主导地位降低了算子多样性——但方向一致。在所有经测试的模型上，经验结果表明校准后的组合保持准确率不低于最佳单算子。这种非劣性，而非任何单一路由增益的大小，是方法论层面的保障：系统设计者可以放心采用校准后的组合，而无需承担性能降至最佳单算子基线以下的风险。

### 6.2 跨基准的外部泛化

**组合方法论能否在不重新设计的情况下迁移至未见过的认知体制？** STALE基准（Chao等人，2026）提供了400个冲突场景及1200个评估查询，涵盖三个维度——状态解决、前提抵抗和隐式策略适应——处于一个不同于MSFT实体-属性结构的对话领域。我们在不重新设计的情况下应用相同的四步方法论：将任务空间分解为认知轴（dim1 → 认知不确定性，dim2 → 冲突解决，dim3 → 时间访问，类型II传播 → 实体组合），相图（第5.2节）选择初始算子，路由器据此派发查询。$\mathcal{S}_T$处理dim1和dim2（含冲突解决前提验证的状态读出），$\mathcal{T}_T$处理dim3（行动建议的时间上下文），$\mathcal{T}_T$用于类型II场景的所有维度（传播链在轨迹中可见）。系统运行于DeepSeek Flash，使用基于规则的关键词裁判。

| 类型 | SR | PR | IPA | 路由 |
|---|---|---|---|---|
| T1 (n=200) | 0.540 | 1.000 | 0.580 | S_T(dim1/2), T_T(dim3) |
| T2 (n=200) | 0.450 | 0.880 | 0.860 | T_T(全部) |
| **全部 (n=400)** | **0.495** | **0.940** | **0.720** | **总体: 0.718** |

组合系统总体达到0.718，而同一基准上CUPMem为0.680，尽管直接比较受基座模型差异（DeepSeek Flash vs GPT-4o-mini）和裁判方法论的干扰。结构性声明独立于此数字：组合收益源于算子互补性，且相图驱动的方法论无需修改即可迁移至外部冲突体制。完整路由策略和失效分类见附录F。

### 6.3 扩展极限与未覆盖的认知区域

**当认知空间超出当前算子基时会发生什么？** ATM-Bench（Mei等人，2026）提供了基于四年个人多模态记忆的1013个QA——其规模和领域超出了受控的MSFT和STALE基准。$\mathcal{S}_T$以检索+KV变体形式评估；主要算子级对比在$\mathcal{R}_T$与$\mathcal{T}_T$之间，其中$\mathcal{T}_T$使用完美轨迹访问，代表认知上限。

| 条件 | 准确率 | 数字类 | 列表类 | 开放式 |
|---|---|---|---|---|
| $\mathcal{R}_T$ | 0.399 | 0.469 | 0.151 | 0.416 |
| $\mathcal{S}_T$ (检索+KV) | 0.175 | 0.172 | 0.094 | 0.198 |
| $\mathcal{T}_T$ (完美轨迹) | **0.518** | 0.622 | 0.144 | 0.547 |

$\mathcal{T}_T$独有覆盖16.3%的查询，$\mathcal{R}_T$独有覆盖3.4%，42.3%未被任何算子覆盖。未覆盖的部分并非方法的失败——它是认知空间超出当前三算子投影基的经验证据。这与框架的设计一致：组合方法论可在识别到其他算子类别时加以容纳（第2节、第8节）。轨迹算子的优势随任务复杂度增长：从单证据查询（n=764）上的+0.092到需要四项及以上证据的查询（n=50）上的+0.320，这与$\mathcal{T}_T$保留时间排序的结构属性一致。

我们强调，ATM-Bench的$\mathcal{T}_T$结果使用完美轨迹访问，代表认知上限。在MSFT（第6.1节）和STALE（第6.2节）上，$\mathcal{T}_T$使用真实会话排序和实体范围游走，方法论得到端到端验证。完整分析（包括Hard-31、投影瓶颈和证据数量梯度）见附录C。

### 6.4 跨模型校准的必要性

**校准是可选的么？** 为检验单一路由表是否足以跨基座模型使用，我们在四个LLM上使用相同协议评估各算子$\mathcal{R}_T$/$\mathcal{S}_T$/$\mathcal{T}_T$（MSFT 112案例，统一DeepSeek Flash裁判）：

| 模型 | R_T | S_T | T_T | 最佳单算子 |
|---|---|---|---|---|
| DeepSeek Flash | 0.679 | 0.384 | 0.661 | R_T 0.679 |
| DeepSeek V4 Pro | 0.839 | 0.625 | 0.786 | R_T 0.839 |
| GPT-4o-mini | 0.741 | 0.527 | 0.812 | T_T 0.812 |
| Claude Sonnet 4 | 0.795 | 0.491 | 0.830 | T_T 0.830 |

算子互补性在所有模型上成立——每个模型在三种算子上展现出不同的准确率分布，确认算子分解无论基座如何均具意义。最优单算子各异：DeepSeek家族模型偏好$\mathcal{R}_T$，而GPT-4o-mini和Claude偏好$\mathcal{T}_T$。唯一算子不变的失效模式是$\mathcal{S}_T$在历史状态和跨实体上的失效（第4.3节）。在八种认知模式中的七种上，每种模式的最佳算子因模型而异（附录E.7）。因此，不存在通用路由表——从某一模型的相图导出的任何路由策略在迁移前都需要经验校准。算子互补性是模型无关的；最优路由则是模型条件化的。这种不对称性直接使得方法论的第二步（第5.3节）成为必要：相图提供结构先验，校准则修正基座依赖的主导偏移。

## 7 讨论与局限性

第4至第6节的结果汇聚为一个设计原则：记忆访问从根本上受到彼此互斥的认知投影的约束，只能通过组合而非单一访问机制的优化来恢复。组合并非集成——每个查询被派发至恰好一个算子，增益源于防止结构上不匹配的配对，而非源于对多个输出的聚合（第6.1节）。相图提供结构先验；校准则解释基座依赖的主导偏移（第6.4节）。在所有经测试的设置下，校准后的组合从未低于最佳单算子，确立了非劣性作为方法论属性，而非经验巧合。

以下局限性反映了当前基于算子的认知访问框架的边界。

**静态路由。** 组合系统采用确定性规则路由，源自逐模式相态分析。能从数据中学习查询到算子映射的动态路由器可适应分布偏移和模型特异性算子优势，有望将路由增益提升至固定规则基线之上。

**算子范围。** 我们识别了三种认知算子，但不声称该集合是穷尽的。我们的主张更为限定：这三种代表了在当前记忆增强型LLM系统中经验上占主导地位的不可约的访问原语（第2节）。其他超出本文研究范围的访问机制可能实例化为不同的认知算子，特别是那些引入可学习的递归、结构化的潜在状态演化或程序化记忆控制的机制。$\mathcal{S}_T$在跨实体查询上的结构弱点（第4.3节）暗示了一个此类候选——跨实体时间轴的关系连接算子。需要系统性地搜索认知轴与访问机制的笛卡尔积以确定闭包。我们强调，本文并不提出记忆算子的完备本体；而是提出一种组合的设计方法论，无论可用的算子有哪些。

**S_T在ATM-Bench上的实现差异。** ATM-Bench上的状态算子采用检索+KV变体，而非具有写入时修订的真实状态投影。真实状态动力学通过受控轨迹对比单独验证（附录H）。

**与ATM-Bench的度量对齐。** 我们的二元LLM裁判与基准针对问题类型的QS评判标准不同。在开放式问题上，两种度量方向一致。

**MSFT-NL范围。** 跨域分析使用300个通过参数化模板自动生成的自然语言案例，含随机实体名称和表层形式。基于模板的生成引入了一个潜在混杂因素：模板结构的系统模式可能膨胀或压缩特定的算子-轴交互。模板内方差未被报告，真实世界自然语言验证仍是未来工作。跨模型验证仅限于结构化MSFT基准；扩展至自然语言域将进一步检验算子相图跨模型的稳定性。

**任务分解的稳健性。** 方法论第一步（将任务空间分解为认知轴）由作者手动完成。该分解在标注者间的可复现性未被量化，且未提出自动化流程。标注者间一致性研究或自动化轴分类方法将增强对该方法论可部署性的信心。

**单一裁判模型。** 所有基准上的所有算子评估均使用DeepSeek Flash作为唯一裁判。在跨模型实验中，GPT-4o-mini和Claude Sonnet 4的答案也由DeepSeek Flash评判。若裁判模型存在系统性偏好（词汇重叠偏差、格式偏好），这些可能相对于其他模型膨胀DeepSeek家族算子的得分。在案例子集上使用替代裁判模型进行交叉裁判验证将量化这一风险。

**T_T完美轨迹与可部署轨迹的差距。** 在ATM-Bench上，$\mathcal{T}_T$使用完美轨迹访问（真实证据ID和会话排序）。在MSFT和STALE上，$\mathcal{T}_T$使用真实会话排序和实体范围游走。ATM-Bench上完美轨迹与可部署轨迹访问之间的差距未被量化；ATM-Bench结果应被解读为认知上限，而非对完整组合方法论在可部署设置中的验证。

**跨实体边界。** 没有算子能可靠地对齐两个实体的时间轴——这是一个已知的理论缺口，促使探索本文识别的三种算子之外的潜在第四种算子类别。

## 8 结论

记忆增强型大语言模型（LLM）智能体必须选择如何访问过去。检索——基于查询条件在记忆库中进行选择——是主导范式，但它仅是至少三种结构性不同的认知算子之一。状态查寻保留事实一致性但舍弃时间结构；轨迹遍历保留时间顺序但牺牲紧凑性。没有单一算子是完备的：每种算子在共享的记忆轨迹上均会引发一种不可恢复的投影失效。基于互不相容投影的组合，通过将每个查询派发到其投影保留了所需认知维度的算子，来恢复认知覆盖。跨模型验证确认，这种互补性是模型无关的，而最优路由则是模型条件化的——状态查寻在所有经测试模型上均于相同的案例上失效；检索与轨迹的偏好随基座LLM而偏移。遵循模型校准路由的组合系统在结构化、自然语言和外部冲突基准上均取得了相对于最佳单算子的一致的增益，同时在所有经测试的设置下经验性地保持了非劣性。将算子基扩展至本文所研究的三种原语之外，实现任务空间分解的自动化，以及在更多基座模型和裁判协议上进行验证，是组合框架在设计上即可容纳的自然的后续步骤。核心启示是：记忆系统设计应被重新定义为基于不可约认知投影的算子组合，而非单一访问范式下的表示优化。

## 附录A：算子定义与系统映射

### A.1 形式算子定义

**定义（检索算子）。** $\mathcal{R}_T(q) = \text{top-}K_\theta(M_T, q)$，其中 $M_T = \{e_1, \dots, e_n\}$ 为无序记忆库，$\text{top-}K_\theta$ 在相似度度量 $\theta$ 下选取与 $q$ 相似度最高的 $K$ 个证据片段。该算子无内部状态，无写入时冲突解决，依赖 $q$ 进行上下文选择。

**定义（状态算子）。** $\mathcal{S}_T(q) = \{ (k, \text{latest}(T, k)) \mid k \in \text{dom}(\mathrm{proj}_s(T)) \}$，其中 $\text{latest}(T, k) = \max_{t} \{ v \mid (k, v, t) \in T \}$。该算子获取上下文与查询无关：返回所有活跃键值对。写入时冲突解决由修订算子 $U$ 执行。

**定义（轨迹算子）。** $\mathcal{T}_T(q) = \text{walk}(G_T, C_q)$，其中 $G_T = (V, E)$ 为转移图，$V = \{e_1, \dots, e_n\}$，$E = \{(e_i, e_{i+1}) \mid 1 \leq i < n\}$，$C_q$ 是从 $q$ 导出的可选约束。最简单形式（MSFT中测试）为 $\text{walk}(G_T)$——对目标实体从 $t=1$ 至 $t=n$ 的无约束遍历。

### A.2 范式对比

| 属性 | $\mathcal{R}_T$ | $\mathcal{S}_T$ | $\mathcal{T}_T$ |
|---|---|---|---|
| 记忆对象 | 记忆库 $M_T$ | 信念状态 $B_t$ | 转移图 $G_T$ |
| 访问 | 查询条件化选择 | 状态读出 | 有序路径游走 |
| 写入时 | 追加 | 修订 $U$ | 追加并索引 |
| 查询时 | 相似度排序 | 无（直接） | 实体范围遍历 |
| 不变性 | 不保证 | 保持（查询无关） | 路径依赖，确定性 |

### A.3 公理覆盖矩阵

| 系统 | 显式全局状态 | 一致性目标 | 原则性修订 | 证据 |
|---|---|---|---|---|
| FadeMem | 是 | 是 | 是 | 含置信阈值的信念衰减 |
| DeltaMem | 是 | 是 | 否 | 追踪增量，无冲突解决 |
| BeliefMem | 是 | 是 | 是 | 含来源可信度的修订 |
| UMA | 否 | 否 | 否 | 自适应检索，无状态 |
| STALE (Chao等人, 2026) | 是 | 是 | 是 | CUPMem：写入时裁决 |
| **本文** | 是 | 是 | 是 | 带修订 $U$ 的 $\mathcal{S}_T$，受控对比 |

路由源自MSFT-NL相图（第5.2节）：dim1（状态解决）映射至认知不确定性 → S_T；dim2（前提抵抗）映射至冲突解决 → S_T + 前提验证器；dim3（隐式策略适应）映射至时间访问 → T_T；类型II传播冲突映射至实体组合 → 全部维度 T_T。

### F.3 失效分类

50场景诊断运行揭示四类失效：评估假象（约40%的报告SR失效，语义正确但关键词不匹配）、上下文不足（约16%，2会话轨迹不足）、行动空间落地（约84%的IPA失效，状态更新未映射到行动空间转换），以及依赖闭包（T2 PR=0.88，跨键传播链未被检出）。

### F.4 与CUPMem对比

| 总体准确率 | 0.680 | 0.718 |
| 基座模型 | GPT-4o-mini | DeepSeek Flash |

## 附录G：STALE与CUPMem分析

*第3节及第6.2节所引用的对CUPMem系统（Chao等人，2026）的独立代码审查与架构分析。*

### G.1 源代码验证

STALE基准代码库（github.com/icedreamc/STALE，MIT许可证）已被独立审查。无硬编码结果；所有准确率计算均使用真实LLM调用。CUPMem通过样本运行验证。

### G.2 CUPMem架构（源自源码）

状态模式：10桶 × 约30条追踪（身份、位置、健康、天气、日常、工作、财务、家庭、偏好、焦点）。追踪携带基数标记和因果链接标记。写入管线：session → chunker → delta_extractor (LLM) → update_resolver (LLM) → invalidation_lanes (LLM 桶桥加潜在提议) → invalidation_judge (LLM) → stale_linker → writer。查询管线：query → track_hints → readout → premise_verifier (LLM) → basis_recovery → action_grounding → answer。CUPMem的传播是LLM驱动的，而非基于规则。

### G.3 与我们的框架的关系

CUPMem是增强的 $\mathcal{S}_T$，具有LLM编排的传播和前提阻断。它未将检索、状态和轨迹区分为不同的认知算子，也未实现算子组合或路由。其贡献在表示层；我们的贡献在访问层。两层互补且独立。

## 附录H：支持性观察

*第4.2节（系统性不稳定、有界状态）及第4.2节（冲突隐藏机制）所引用的次要实验发现。*

### H.1 算子动力学（多轨迹受控对比）

四条轨迹（T0至T3），证据结构相同但实体和领域不同，遵循9步证据协议。检索证据以top-3上下文提供；状态使用置信阈值0.8的修订算子 $U$。第9步时，$\mathcal{R}_T$ 废弃率：0.39±0.14（范围0.286至0.571）。$\mathcal{S}_T$ 废弃率：全部四条轨迹均为0.00±0.00。

检索算子展现出显著的轨迹依赖方差：检索排序对查询与证据文本间的词汇重叠敏感，而词汇重叠因领域而异。专业领域轨迹（通用词汇）产生最高废弃率（0.571）；体育领域轨迹（领域特化词汇）产生最低（0.286）。这种领域依赖方差是 $\mathcal{R}_T$ 的结构属性：其输出取决于查询-证据的词汇几何，该几何不在系统设计者的控制范围内。状态算子在所有轨迹上收敛至零废弃率，无论轨迹领域或证据到达顺序如何。

### H.2 冲突隐藏

单轨迹杀手级追踪揭示了检索无法解决写入时冲突的机制。第9步，一条显式更正（"Alice不再住在西雅图。她已于2025年永久迁至芝加哥"）在 $\mathcal{R}_T$ 检索输出中排名第八，位于四条过时片段之后。Jaccard分析显示，该更正与查询共享相同的词汇重叠（{alice}），但包含18个额外词汇，使Jaccard分母膨胀3.25倍，导致其得分低于更短、信息稀疏的过时片段。这不是检索质量问题——而是相似度选择的固有结构属性：更长、信息更丰富的证据被按文档长度归一化的度量系统性惩罚。正确与过时的证据在LLM的上下文窗口中永不共现，使得读取时的冲突解决在结构上不可能。

### H.3 有界状态验证

当状态提取由LLM从非结构化文本执行（无置信度校准）时，废弃率为3.85%，而扁平检索为19.23%——提升5倍。提取误差有界（约0.04），不随记忆体量增大而复合，因为每次提取独立；检索误差无界，因为排序竞争随记忆库增长而加剧。
